# 05_S1mpleYOLO
A minimal YOLO implementation that divides an image into an S x S grid (S=20) 
and predicts a single bounding box per grid cell. Each training image contains 
exactly one circle. The model outputs 5 channels per grid cell:
- tx, ty: offset within grid cell (after sigmoid, range 0-1)
- tw, th: width and height (after sigmoid, range 0-1, scaled by IMG_SIZE)
- conf_logits: confidence score (sigmoid applied to get probability)

## Grid-Based Detection
- 640*640 Image downsampled into S×S grid (S=20), an S×S feature map with high number of channels
- Each grid cell (all channels in the specific pixel) predicts one bounding box
- Only grid cells containing object center are "positive" samples  

This is True for SimpleYOLO only, *the noobest YOLO possible*. Modern real-world YOLO detectors implement more advanced techniques. Ask an LLM if you are curious.

## Encoding/Decoding
- **tx, ty**: Offset within grid cell (0-1), decoded as `cx = (grid_x + tx) / S * IMG_SIZE`
- **tw, th**: Relative size, decoded as `w = tw * IMG_SIZE`, `h = th * IMG_SIZE`
- **conf**: Confidence that object exists in this grid cell

## Batch Normalization
`BatchNorm2d` normalizes data into variance=1 then applies a learnable scale (γ) and shift (β). It lets you train deeper CNNs much faster, with higher learning rates, less careful tuning, and often without needing Dropout by ensuring every layer receives consistently scaled inputs.  
It needs a higher batch size (>= 8) to work perfectly.

## Model Structure

| 1 | Conv2d + BN + LeakyReLU | 3×3 | 2 | 320×320×16 → 160×160×32 | 4,640 | Downsample |
| 2 | Conv2d + BN + LeakyReLU | 3×3 | 2 | 160×160×32 → 80×80×64 | 18,496 | Downsample |
| 3 | Conv2d + BN + LeakyReLU | 3×3 | 2 | 80×80×64 → 40×40×128 | 73,856 | Downsample |
| 4 | Conv2d + BN + LeakyReLU | 3×3 | 2 | 40×40×128 → 20×20×256 | 295,168 | Downsample to grid resolution |
| 5 | Conv2d + BN + LeakyReLU | 3×3 | 1 | 20×20×256 → 20×20×512 | 1,180,160 | Feature processing (no downsampling) |
| 6 | Conv2d (detection head) | 1×1 | 1 | 20×20×512 → 20×20×5 | 2,565 | Output: tx, ty, tw, th, conf_logits |

**Total Parameters:** 1,575,333 (1.58M)
- Conv layers: 1,575,333
- BatchNorm layers: 1,328 (2× parameters per BN layer: 16×2 + 32×2 + 64×2 + 128×2 + 256×2 + 512×2 = 1,328)
- **Total:** 1,575,333 + 1,328 = **1,576,661**

**Model Size:** ~6.3 MB (float32)  

The SimpleYOLO model is a fully convolutional neural network designed for single-object detection with a fixed 20×20 grid output.  
The model consists of 8 convolutional layers that progressively downsample the input image from 640×640 to 20×20 spatial resolution. Each downsampling step is achieved using stride=2 convolutions, reducing the feature map size by half (640→320→160→80→40→20). After reaching the final grid resolution, two additional stride=1 convolutions (256→512 and 512→5) process the features without changing spatial dimensions.  
Given an input of shape (B, 3, 640, 640), the model outputs (B, 5, 20, 20) — a compact representation where each of the 400 grid cells predicts exactly one bounding box with its 5 parameters. This minimal design makes the model easy to understand while capturing the core YOLO philosophy of grid-based detection.

## Loss Function
The SimpleYOLO loss consists of two main components: **coordinate loss** and **confidence loss**, carefully weighted to balance localization accuracy and detection reliability.

### Coordinate Loss (Localization or Box_Loss)
Describes how inaccurate the bounding box is.  
Computed using Mean Squared Error (MSE) on the four bounding box parameters: (tx, ty, tw, th). Applied only to positive targets. 
For 1 target, it looks like `tx_err ^ 2 + ty_err ^ 2 + tw_err ^ 2 + th_err ^ 2`.

### Confidence Loss (Objectness)
Describes how inaccurate the judgement about positive/negative is. 
Each cell predicts a confidence score indicating whether an object exists at that location. To handle the severe class imbalance (only ~1 positive cell out of 400 per image), positive cells are weighted 1.0 (full loss), while negative cells are weighted 0.5 (half loss). This reduces the impact of the many negative cells while still suppressing false positives.
For 1 target (cell), it looks like `-log(conf)` (positive cell) or `-log(1 - conf)` (negative cell).  
The total confidence loss for one image is **weighted average** over all 400 cells.

### Final Loss
The total loss is the sum of coordinate loss and confidence loss, with no separate classification loss since this is a single-class (circle) detection task. The loss is computed per batch and backpropagated to update the model parameters during training.

## NMS (Non-Maximum Suppression)
A post-processing technique used to **eliminate duplicate detections** of the same object.  
Since YOLO styled models predicts one bounding box per grid cell (20×20 = 400 boxes total), many adjacent cells may predict overlapping boxes with high confidence around the same circle. NMS keeps the box with highest confidence, while deleting all other duplicates.  
IoU (Intersection over Union) measures the overlap between two bounding boxes. Formula: `IoU = (A ∩ B) / (A ∪ B) = Intersection / (Area_A + Area_B - Intersection)`
- **IoU = 1.0**: Perfect overlap (identical boxes)
- **IoU = 0.5**: Moderate overlap (typical NMS threshold)
- **IoU = 0.0**: No overlap at all  

In NMS, boxes with IoU > 0.5 are considered duplicates and removed.  
Procedure: 
1. Sort all boxes by confidence (highest to lowest)
2. Pick the highest-confidence box and add it to final list
3. Remove all boxes with IoU > threshold (e.g., 0.5) with the picked box
4. Repeat until no boxes remain  

Time Complexity: `O(n ^ 2)` in worst case.  
- Sort = `O(n log n)`.
- IoU comparing = `O(n ^ 2)` (Each selected box compares with all remaining boxes). 
- Adding to `O(n ^ 2)` total.
